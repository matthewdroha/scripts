Filename: getTCver.pl Project: ihdk Original Author: Matthew Roha

DESCRIPTION

getTCver.pl prints out collateral versions for a given UE session. Input is a tool configuration or list of configurations. Output is a .csv file. Script works over dot process names and is useful when there is a need to collect information on large number of configs.

SYNOPSIS

getTCver.pl
 --snapdir <DART snapshot archive directory>
 --targz <IRR compressed tarball>
 [--run-name <logfile prefix>]
 [--env 'VAR=VALUE']
 [--help] [--verbose] [--debug]

flag descriptions:

--config

MIG tool collateral configuration name (example: mig74p0cnl1b0polo)

--skipheader

Do not print csv first row labels. Handy if concatenating files.

--run-name

Optional. String label prefix for output files only.

--env

Optional. Set env var at start of execution. Can provide more than one --env flag. Format is VAR=VALUE

--debug

Run flow in debug mode. Temporary files are not deleted and additional data is placed in log file.

--verbose

Will add status messages to STDOUT.

--help

This usage message will appear.

example:

$exe_name --config mig74p0cnl1b0polo

Files that result from this run:
 $PWD/<CONFIG>.getTCver.log
 $PWD/<CONFIG>.getTCver.csv

PROGRAM FLOW

Inputs

Flow

- Stage 1: Run setproj and check if config is valid. - Stage 2: Run mig ihdk setup to build session tool/collateral list - Stage 3: Parse tool/collateral list history with release comments. Make note of current config. - Stage 4: For each tool version, get date installed from MIG CAD tree and HDK CAD tree - Stage 5: Build .csv file.

CSV FIELDS

REPORT ORDER -> Original record order in case engineer wants to restore original order after Excel filtering YYYYWW -> Intel workweek when transaction occurred TRANSACTION COUNT FOR WW -> Total number of transactions (rows) recorded in given workweek. 0 is a placeholder TRANSACTION TYPE -> The nature of the transaction (see below for possible values) TRANSACTION DATE -> Date and time of transaction TOOL COLLATERAL TYPE -> The type of tool or collaterl (see below for possible values) PROCESS -> Process technology CONFIGURATION -> MIG tool collateral configuration name TOOL COLLATERAL -> Tool or collateral name VERSION -> Tool or collateral version name REVISION -> Integer representing a uniquer revision number for tool collateral change (incrememted chronologically) IS CURRENT REVISION -> YES or NO representing whether change exists in current tool collateral version

TRANSACTION TYPE CONSTANTS

CONFIG_TC_VERSION_CHANGE -> Tool collateral tool version change occurred inside given configuration (including initial) MIG_CAD_TV_VERSION_CHANGE -> Tool collateral version update inside installed MIG CAD tree version.

TOOL COLLATERAL TYPE CONSTANTS

TOOL -> CAD tool COLLATERAL -> Technology collateral
