Comments regarding this release:

This is Matanuska Glacier ALP RTL1.0 release. Following are some of the notable changes/issues addressed from RTL0.8:

·         PD scheme has been redone to replace LDOs with PG. the details are documented in an addendum document specific to test chip. HAS is not updated as its being maintained for the product.

·         Updated bump map/lef already provided to test chip team

·         Link to Power spec addendum document.

·         https://vthsd.intel.com/hsd/seg_ip/default.aspx#document/collaborate.aspx?dynamic=yes&query_name=%3CDoc%20Folder%3E&query_id=11239387

·         Both the MG instances inside the super family will be now DCO based. The family and instance names have not been updated to make this change seemless to the integration teams. The HAS/IG PDF are not updated yet to reflect this.

·         Back channel EQ is enabled and details are outlined in the notes below.

·         Scan is not supported

·         stap, IDV, bscan connectivity issues highlighted in RTL0.8 are fixed.

·         Others issues related to fsdbsuppress/compile issues are addressed.

·         Libs are provided for corners 9, 14, 15

·         PY/spice netlists are not provided at this time – please contact IP lead if needed.

·         SoC/Phy interface pins have no changes wrt RTL0.8

Procedure:

1.      Clear cri_use_fs32 = 1’b0 to use fs63 (we need all the precision we can get)

2.      Set cri_txdeemph_override[17:0] = 6’d63

3.      Set cri_txdeemph_override_en = 1’b1

4.      Wait a few clock cycles

5.      Set cri_calcinit = 1’b1
